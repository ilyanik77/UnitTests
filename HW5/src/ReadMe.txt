Задание 1.
  Представьте, что вы работаете над разработкой простого приложения для записной книжки,
которое позволяет пользователям добавлять, редактировать и удалять контакты.
Ваша задача - придумать как можно больше различных тестов (юнит-тесты, интеграционные тесты, сквозные тесты)
для этого приложения. Напишите название каждого теста, его тип и краткое описание того, что этот тест проверяет.

Решение.
  Юнит-тесты:

1.AddUserTest - проверяет добавление нового контакта.
2.UpdateUserTest - проверяет редактирование контакта.
3.DeleteUserTest - проверяет удаление контакта.
4.SearchUserByNameTest - поиск контакта по имени.
5.SearchUserByPhoneTest - поиск контакта по номеру телефона.

  Интеграционные тесты:

1.PrintUserByNameTest - вывести данные контакта по имени.
2.PrintUserByPhoneTest - вывести данные контакта по номеру телефона.

  Сквозной тест:

1.ApplicationTest - проверяет весь функционал приложения


Задание 2.
  Ниже список тестовых сценариев. Ваша задача - определить тип каждого теста
(юнит-тест, интеграционный тест, сквозной тест) и объяснить, почему вы так решили.
1.Проверка того, что функция addContact корректно добавляет новый контакт в список контактов"".
2.Проверка того, что при добавлении контакта через пользовательский интерфейс, контакт корректно
  отображается в списке контактов"".
3.Проверка полного цикла работы с контактом: создание контакта, его редактирование и последующее удаление"".

Решение.

1.Юнит-тест. - нет внешних зависимостей и нет взаимодействия между другими модулями(классами).
2.Интеграционный тест. - взаимодействие с другими модулями(классами).
3.Сквозной тест. - проверка полного цикла работы и взаимодействия между модулями.
